# Security Policy

## Supported Versions

目前 **Nexus** 项目仅对最新版本提供安全更新支持。请确保你正在使用下表中的受支持版本。

| Version | Supported | Notes |
| --- | --- | --- |
| **v1.1.x** | :white_check_mark: | **推荐版本**。包含最新的功能与安全修复。 |
| **v1.0.x** | :x: | **已弃用**。存在已知风险或不再维护，请升级至 v1.1.0。 |

---

## Reporting a Vulnerability (报告漏洞)

**请不要通过公开的 GitHub Issues、Discussions 或 Pull Requests 提交安全漏洞。**

为了保护广大用户，我们建议通过私下渠道报告潜在的安全问题。

### 1. 如何提交报告

如果你发现了安全漏洞（如 SQL 注入、越权访问、RCE 等），请直接发送邮件至：

* **联系邮箱：** `yakit@knowbul.dpdns.org`

### 2. 报告应包含的内容

为了让我们能快速复现并修复问题，请在邮件中包含以下信息：

* **漏洞类型**：例如“越权访问”、“敏感信息泄露”等。
* **影响版本**：你在哪个版本中发现了该问题（例如 v1.1.0）。
* **复现步骤 (PoC)**：详细的操作步骤、测试脚本或请求包截图。
* **潜在影响**：该漏洞如果被利用，会导致什么后果。

### 3. 我们的处理流程

1. **确认收悉**：我们会在 **48 小时** 内回复邮件，确认已收到你的报告。
2. **漏洞修复**：确认漏洞存在后，我们会将其列为高优先级任务进行修复。
3. **公开致谢**：在漏洞修复并发布新版本后，如果你愿意，我们会在 Release 说明中对你进行致谢。

---

## Responsible Disclosure Policy (负责任的披露政策)

我们承诺会尽快处理每一个有效的漏洞。同时也希望报告者遵守以下原则：

* 在漏洞修复并发布新版本之前，请勿向外界披露漏洞细节。
* 避免对生产环境或他人数据进行破坏性的测试。
* 我们遵循“负责任的披露”原则，不会对遵循本政策的漏洞发现者采取法律行动。
